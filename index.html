Dokumentasi lengkap arsitektur, logika kerja, dan kredit pengembang sistem

☀ Apa Ini?

WEB MONITOR PLTS adalah panel monitoring & kontrol jarak jauh untuk sistem Pembangkit Listrik Tenaga Surya (PLTS) berbasis mikrokontroler ESP32. Web ini dirancang sebagai jembatan antara perangkat keras di lapangan — baterai, panel surya, jaringan listrik PLN, dan empat relay beban — dengan pengguna di mana pun berada, selama masih terhubung internet. Setiap perubahan tegangan, arus, daya, hingga status relay dibaca dan ditampilkan hampir seketika (near real-time), sehingga kondisi sistem kelistrikan dapat dipantau dan dikendalikan tanpa perlu berada di lokasi fisik panel.

⚙ Arsitektur & Alur Data Sistem

Inti sistem adalah papan ESP32 yang bertindak sebagai web server sekaligus otak pengendali. ESP32 membaca dua sensor tegangan (modul pembagi tegangan 25V) untuk memantau tegangan baterai dan tegangan panel surya secara langsung melalui pin analog, lalu membaca data listrik AC — tegangan, arus, daya, frekuensi, power factor, dan akumulasi energi — dari modul PZEM-004T V4 melalui komunikasi serial. Seluruh data ini diolah, ditampilkan di halaman web, dan sekaligus menjadi dasar pengambilan keputusan logika otomatis untuk menyalakan atau mematikan empat jalur relay beban.

Front-end web ini di-hosting terpisah melalui GitHub Pages, lalu berkomunikasi dengan ESP32 di jaringan lokal melalui terowongan (tunnel) SSH yang menjembatani akses dari luar jaringan rumah ke server ESP32, sehingga panel tetap bisa diakses dari mana saja tanpa IP publik permanen. Sebagai pelengkap, sebuah papan NodeMCU ESP8266 terpisah menggerakkan layar OLED kecil yang menampilkan ringkasan status — tegangan baterai, level baterai, tegangan AC, dan jumlah pengunjung — sebagai tampilan fisik cepat di dekat panel itu sendiri.

📡 TAB MONITORING
Menampilkan seluruh data sensor: tegangan & arus baterai, tegangan panel surya, level baterai, energi harian, serta tegangan/arus/daya/frekuensi/energi listrik AC dari PZEM-004T. Semua data ini otomatis diperbarui setiap 2 detik — tidak perlu login untuk melihatnya.
TAB KONTROL
Tempat mengatur jadwal timer otomatis dan menyalakan/mematikan 4 relay secara manual. Karena bisa mengubah kondisi perangkat fisik, tab ini mewajibkan LOGIN admin (tombol LOGIN di kanan atas header) sebelum tombol/toggle bisa digunakan.
⚙ CONTROL PANEL ADMIN
Panel khusus admin untuk melihat status ESP32 (uptime, RAM, WiFi, IP), mengganti password, mengatur kalibrasi sensor, menetapkan limit biaya listrik, batas arus AC (proteksi overcurrent), serta melihat log akses login. Panel ini punya password sendiri, terpisah dari password LOGIN untuk kontrol relay.
ℹ TAB TENTANG
Halaman yang sedang kamu baca ini — dokumentasi menyeluruh cara kerja sistem, logika tiap relay, mekanisme keamanan, hingga profil di balik pengembangan web ini.

⏰ Logika Otomatis Tiap Relay

  • R1 — INVERTER [Manual + PZEM]: otomatis ON saat PLN tidak ada/lemah (<50V), OFF saat PLN ada (≥50V). Bisa dioverride manual, dan akan tetap MANUAL sampai tombol AUTO ditekan kembali.
  • R2 — LAMPU UTAMA [Manual | Timer + PZEM]: ON otomatis selama jam timer aktif, ATAU saat PLN tidak ada. Bisa dioverride manual, tetap MANUAL sampai tombol AUTO ditekan.
  • R3 — OUTPUT AC [Manual + Auto Timer]: ON otomatis selama jam timer aktif saja. Bisa dioverride manual, tetap MANUAL sampai tombol AUTO ditekan.
  • R4 — LAMPU PANEL [Manual sesaat + PZEM]: ON saat PLN 0V, OFF saat PLN ≥50V. Override manual hanya berlaku sesaat — otomatis kembali ke mode PZEM dalam ±1 detik.

Prinsip di balik seluruh logika ini sederhana: sistem selalu berusaha kembali ke mode AUTO sebagai kondisi normal, sementara mode MANUAL hanya menjadi pengecualian sementara yang sepenuhnya berada di bawah kendali admin, bukan sebaliknya.

🌡 Akurasi Sensor & Kalibrasi

Karena setiap komponen elektronik memiliki toleransi resistansi dan non-linearitas pembacaan ADC, tegangan baterai dikalibrasi menggunakan lookup table 5 titik (pasangan nilai mentah & nilai aktual) supaya kurva pembacaan tetap presisi di seluruh rentang tegangan, bukan hanya pada satu titik kalibrasi. Tegangan panel surya menggunakan faktor kalibrasi tunggal yang dapat disesuaikan langsung dari Control Panel Admin. Kalibrasi ini tersimpan di memori ESP32 sehingga tidak hilang saat perangkat dinyalakan ulang.

🔒 Dua Jenis Login yang Berbeda

Web ini memakai dua sistem password terpisah supaya lebih aman:

  • LOGIN (tombol header) — dipakai untuk mengontrol relay & mengubah jadwal timer di tab Kontrol.
  • Password Control Panel — dipakai khusus untuk membuka tab Control Panel Admin. Login di salah satu tidak otomatis membuka yang lain.

💰 Proteksi Biaya & Arus

Admin dapat menetapkan Limit Biaya (Rupiah), dihitung otomatis dari akumulasi energi AC berdasarkan tarif per kWh — jika biaya akumulasi menyentuh limit ini, semua relay otomatis dikunci OFF sampai limit dinaikkan kembali. Admin juga dapat menetapkan Batas Maksimal Arus AC — jika terlampaui, semua relay langsung TRIP OFF layaknya sebuah circuit breaker (MCB), dan tetap OFF sampai direset manual di Control Panel, walau arus yang mengalir sudah kembali normal. Kedua mekanisme ini berjalan independen dari logika otomatis relay, sehingga tetap aktif melindungi sistem meskipun relay sedang dalam mode manual sekalipun.

⚡ Keandalan Antarmuka

Untuk menjaga pengalaman yang mulus meski di jaringan yang tidak stabil, tampilan tombol relay memakai teknik optimistic UI disertai penguncian sementara per-relay (±3 detik) setelah ditekan — supaya status di layar tidak berkedip atau tertimpa balik oleh data polling sebelum perintah benar-benar diproses ESP32. Setiap aksi penting juga disertai notifikasi singkat (toast) di bagian bawah layar, dan seluruh percobaan login dicatat dalam log akses yang dapat ditinjau admin.
